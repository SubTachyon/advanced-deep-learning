# Written Summary

I had a pretty annoying time just getting the `torch_geometric` stuff to work, so by the time everything finally ran I was honestly a bit demotivated and did not give this exercise my absolute best effort. For this exercise I used a graph neural network to predict the neutrino interaction position in the detector, so the target was the (x, y) point for each event. I loaded the train, validation, and test parquet files and turned each event into one graph, where each photon became a node with three features: time, x position, and y position. I normalized these features using only the training set. The model itself was a pretty simple GNN with three `DynamicEdgeConv` layers, then global mean pooling, then a small MLP that outputs the predicted (x, y) position. I trained it with MSE loss for 30 epochs using Adam.

I also made a few plots at the end: the training and validation loss curve, the true vs predicted positions in the full plane, predicted vs true separately for x and y, and a couple of residual plots for x. The model did learn something real. In the predicted-vs-true plots, both x and y followed the diagonal reasonably well, so it was not just outputting nonsense. The overall position plot also showed that the predictions were landing in the right general regions. At the same time, the scatter was still pretty noticeable, so the reconstruction is definitely not perfect. The x residual plots were useful for checking whether there was some obvious bias or shift. I would not call the result amazing, but it worked well enough to show that the GNN setup can extract useful position information from the photon data.

The biggest challenge was honestly the setup. Getting the `torch_geometric` packages installed and working was more annoying than the actual modeling, and that kind of killed my momentum early on. Another limitation is that I kept the model pretty basic. I did not do much tuning beyond trying a different value of `k`, and I also did not really explore other graph architectures, pooling methods, or loss choices. The diagnostics were also a bit uneven since I only made residual plots for x and not for y. If I were improving this, I would spend more time tuning the architecture, maybe try a few different graph conv layers, add better diagnostics for both coordinates, and probably save the best validation model instead of just taking the final epoch.
